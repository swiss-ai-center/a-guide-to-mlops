Mission briefing

Why this guide?

Building a model is only the beginning. This guide is a step-by-step flight plan that takes you from a notebook experiment to a reproducible, monitored, and continuously-improved ML system. We use concrete tools — DVC, CML, Docker, BentoML and Label Studio — so you can follow along with real code.

{% include ".icons/lucide/chevron-down.svg" %} Next stop {% include ".icons/lucide/book-open.svg" %} Read the concept
Part 1 — Lift-off

Local training & model evaluation

Start on solid ground. Version your data with DVC, track experiments with CML, and build a reproducible training pipeline that you can run on your own machine before you ever think about the cloud.

{% include ".icons/lucide/chevron-down.svg" %} Next stop {% include ".icons/lucide/flask-conical.svg" %} Train locally
Part 2 — Escape velocity

Move the model to the cloud

When your local machine is no longer enough, move the pipeline to the cloud. Use CML to automate training in CI, store artifacts in S3, and keep every experiment reproducible no matter where it runs.

{% include ".icons/lucide/chevron-down.svg" %} Next stop {% include ".icons/lucide/cloud.svg" %} Go cloud
Part 3 — In orbit

Serve & deploy the model

A trained model only creates value when it can serve predictions. Package it with BentoML, expose a REST API, deploy behind a reverse proxy, and add monitoring so you know when performance starts to drift.

{% include ".icons/lucide/chevron-down.svg" %} Next stop {% include ".icons/lucide/server.svg" %} Deploy
Part 4 — Deep space loop

Label data & retrain

Production data changes. Set up a labeling workflow with Label Studio, review edge cases, merge new labels back into your dataset, and trigger a new training run without losing reproducibility.

{% include ".icons/lucide/chevron-down.svg" %} Next stop {% include ".icons/lucide/tags.svg" %} Retrain
Reentry & cleanup

Bring it safely back to Earth

The journey ends with a controlled landing. Tear down cloud resources, remove containers and clusters, archive artifacts, and leave your environment clean. A safe re-entry is the last step of a successful MLOps mission.

Ready for launch?

The guide is open source and built to be followed hands-on. Pick a chapter, launch your terminal, and start shipping ML systems.
